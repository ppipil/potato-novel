## Context

当前 Potato Novel 的书城模板故事和自由创作故事共用同一套 story package 生成链路：进入前可能触发骨架生成、节点补全、预热与缓存状态管理，导致书城故事也承担了运行时生成成本。用户因此难以理解书架右侧状态代表的是“加载已有内容”还是“重新生成新内容”，自由创作与书城模板也共享了相同的慢路径。

现有实现已经具备数据库存储、书架页、阅读页和两阶段 story package 生成逻辑，但这些能力尚未按内容来源分层。新的设计需要在保留现有 OAuth 和本地阅读流程的前提下，把“公共故事分发”和“个性化故事生产”拆开。

## Goals / Non-Goals

**Goals:**
- 将书城故事定义为数据库中的固定 story package，并允许所有用户读取相同底稿。
- 将自由创作定义为唯一需要运行时生成 story package 的入口，并要求生成状态可感知。
- 将运行时的 SecondMe 调用从“整包生成”收缩到“自由创作内容生产 + 游玩中的人格判断与解读”。
- 重塑书架 UI 心智，使书城卡片展示“加载/进入”状态，自由创作区域展示“生成”状态。
- 为后续性能优化建立清晰边界：书城追求秒开，自由创作追求首幕可进入。

**Non-Goals:**
- 不在本次设计中定义完整的编辑后台或运营工具。
- 不在本次设计中要求立刻实现流式 token 打字机效果。
- 不在本次设计中重写 OAuth 或更换会话机制。

## Decisions

### 1. 书城故事与自由创作故事分开建模
书城故事将被视为“已发布内容”，从数据库读取完整 story package；自由创作故事将被视为“用户生成内容”，从输入开头开始触发生成任务。

选择该方案而不是继续复用同一条 preload/hydrate 链路，是因为书城的首要目标是稳定与速度，而自由创作的首要目标是个性化与可见进度。这两个目标天然不同。

### 2. 书城故事进入时不重新生成骨架或正文
书城入口只负责加载和进入，不负责生成。用户进入书城故事时，前端直接读取数据库中的 package，必要时仅构建本地阅读状态。

选择该方案而不是“进入时为每个用户复制一份 package 并补全节点”，是因为书城故事本质是公共内容，不应把 SecondMe 的生成成本绑定到每次消费上。

### 3. 自由创作采用分阶段生成状态
自由创作流程将显式分为骨架、开场、首层分支、可进入等阶段，并允许后续节点后台补全。前端显示真实阶段和预计耗时，而不是沿用书城的下载心智。

选择阶段化状态而不是百分比伪进度，是因为自由创作耗时来自离散任务，更适合展示“正在做什么”，也更方便后端演进为轮询或流式状态接口。

### 4. 游玩中的 SecondMe 调用改为轻量人格判断
在固定故事 package 中游玩时，SecondMe 主要负责给出“更可能的选择倾向、理由、人格解释或尾声解读”，而不是在进入故事时重生成整个故事包。

选择该方案而不是继续在书城中运行时补齐全文，是因为用户游玩的个性化需求主要体现在“我会怎么选”而非“故事为我重新写一遍”。

### 5. 书架 UI 按内容来源拆心智
书城卡片只展示“进入 / 加载中 / 可进入”一类状态，自由创作区域展示“生成骨架 / 补全开场 / 准备分支 / 可进入”等状态。书城中应移除“下载 / 重新下载 / 预缓存”等容易误导的词汇。

选择该方案而不是继续优化旧下载状态，是因为旧状态设计服务于即将退场的架构，不值得继续深修。

## Risks / Trade-offs

- [书城内容需要预先准备] → 需要新增一套内容入库或迁移流程；短期可以先手工导入少量书城故事。
- [自由创作仍然会慢] → 通过首幕可进入、阶段状态、后台补全来降低等待焦虑，而不是承诺瞬时生成。
- [两种故事来源并存会增加前端分支判断] → 用明确的数据字段区分 content source，避免 UI 通过启发式判断。
- [人格判断可能和玩家真实选择不一致] → 将该能力设计为“推荐与解释”，而不是自动替用户执行。

## Migration Plan

1. 在数据库中引入书城固定 story package 的来源标识与读取接口。
2. 将现有模板故事入口从“预下载生成”切换为“直接加载固定 package”。
3. 将自由创作入口保留为生成链路，并增加状态阶段模型。
4. 为游玩过程增加人格判断接口，但不替换现有手动选项机制。
5. 逐步移除旧的下载/预缓存文案与依赖，保留必要兼容层直到新入口稳定。

回滚策略：如果新分流上线后出现严重问题，可以暂时保留自由创作生成链路不变，仅回退书城入口的 UI 和接口映射；数据库中新增的固定故事数据不会影响旧 session 结构。

## Open Questions

- 书城固定 story package 的生产流程由人工导入、后台工具导入，还是由自由创作转正导入？
- 人格判断接口是只返回推荐选项，还是同时返回解释、置信度和展示文案？
- 自由创作的预计时间是写死分阶段文案，还是需要后端基于任务状态计算近似剩余时间？
